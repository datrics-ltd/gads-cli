### 2026-03-18 — Phase 1.2 Configuration
- `cmd/config_cmd.go` — `gads config set|get|list|path` subcommands
  - `set`: reads existing YAML, updates key, writes back with 0600 perms
  - `get`: reads effective value via viper (flag > env > file > default)
  - `list`: reads config file + active GADS_ env vars, redacts sensitive keys (token/secret/password)
  - `path`: prints `~/.gads/config.yaml` path
- Auto-create `~/.gads/` on first use (already in config.Init)
- Env var precedence (GADS_ prefix) already wired in Phase 1.1; this phase validates it works

### 2026-03-18 — Phase 1.3 Authentication — OAuth2
- `internal/auth/credentials.go` — NEW: Credentials struct, Load/Save/Delete, CredentialsPath(); writes to `~/.gads/credentials.json` with 0600 perms
- `internal/auth/oauth.go` — REWRITTEN: full OAuth2 flow
  - `TokenSource` interface (preserved)
  - `OAuthTokenSource` — implements TokenSource, auto-refreshes on expiry via stored refresh token
  - `Login()` — starts local HTTP server on random port, opens browser via `github.com/pkg/browser`, waits for callback with CSRF state check, exchanges code for tokens, persists to credentials file
  - `Logout()` — best-effort token revocation at Google, deletes local credentials
  - `ForceRefresh()` — refreshes access token immediately using stored refresh token
  - `GetStatus()` — returns AuthStatus (logged in, token expiry, dev token / client_id configured)
- `cmd/auth.go` — NEW: `gads auth login|status|logout|refresh` subcommands
  - `login`: calls auth.Login(), prints expiry and credentials path
  - `status`: shows developer token, client_id, and login state; detects GADS_ACCESS_TOKEN env var
  - `logout`: calls auth.Logout(), confirms deletion
  - `refresh`: calls auth.ForceRefresh(), shows confirmation (debug token preview with --verbose)
- Dependencies added: `golang.org/x/oauth2`, `golang.org/x/oauth2/google`, `github.com/pkg/browser`

### 2026-03-18 — Phase 1.4 API Client
- `internal/api/client.go` — REWRITTEN: full HTTP client implementation
  - `Client` struct with `TokenSource` interface, developer token, customer ID, retries, verbose
  - `NewClient(Config)` — config struct pattern for clean construction
  - `Do(*http.Request)` — executes request with body replay on retry, auth header injection, verbose logging
  - `Get(path, headers)` / `Post(path, body, headers)` — convenience methods
  - `injectAuthHeaders()` — injects `developer-token` + `Authorization: Bearer` headers; auto-refreshes via TokenSource
  - Retry loop with exponential backoff (1s, 2s, 4s) for 429/500/503 responses
  - Verbose mode logs full request (method, URL, headers with redaction, body) and response (status, timing) to stderr
- `internal/api/errors.go` — NEW: API error parsing and mapping
  - `ParseAPIError(*http.Response)` — parses Google Ads REST error envelope into `APIError`
  - Maps known error types (AUTHENTICATION_ERROR, AUTHORIZATION_ERROR, QUOTA_ERROR, QUERY_ERROR, REQUEST_ERROR) to human-readable messages
  - Falls back to HTTP status mapping for unstructured errors
- `cmd/root.go` — added `--retries` flag (default 3), bound to viper key `retries`
