### 2026-03-18 ‚Äî Phase 1.3 Authentication ‚Äî OAuth2
- `internal/auth/credentials.go` ‚Äî NEW: Credentials struct, Load/Save/Delete, CredentialsPath(); writes to `~/.gads/credentials.json` with 0600 perms
- `internal/auth/oauth.go` ‚Äî REWRITTEN: full OAuth2 flow
  - `TokenSource` interface (preserved)
  - `OAuthTokenSource` ‚Äî implements TokenSource, auto-refreshes on expiry via stored refresh token
  - `Login()` ‚Äî starts local HTTP server on random port, opens browser via `github.com/pkg/browser`, waits for callback with CSRF state check, exchanges code for tokens, persists to credentials file
  - `Logout()` ‚Äî best-effort token revocation at Google, deletes local credentials
  - `ForceRefresh()` ‚Äî refreshes access token immediately using stored refresh token
  - `GetStatus()` ‚Äî returns AuthStatus (logged in, token expiry, dev token / client_id configured)
- `cmd/auth.go` ‚Äî NEW: `gads auth login|status|logout|refresh` subcommands
  - `login`: calls auth.Login(), prints expiry and credentials path
  - `status`: shows developer token, client_id, and login state; detects GADS_ACCESS_TOKEN env var
  - `logout`: calls auth.Logout(), confirms deletion
  - `refresh`: calls auth.ForceRefresh(), shows confirmation (debug token preview with --verbose)
- Dependencies added: `golang.org/x/oauth2`, `golang.org/x/oauth2/google`, `github.com/pkg/browser`

### 2026-03-18 ‚Äî Phase 1.4 API Client
- `internal/api/client.go` ‚Äî REWRITTEN: full HTTP client implementation
  - `Client` struct with `TokenSource` interface, developer token, customer ID, retries, verbose
  - `NewClient(Config)` ‚Äî config struct pattern for clean construction
  - `Do(*http.Request)` ‚Äî executes request with body replay on retry, auth header injection, verbose logging
  - `Get(path, headers)` / `Post(path, body, headers)` ‚Äî convenience methods
  - `injectAuthHeaders()` ‚Äî injects `developer-token` + `Authorization: Bearer` headers; auto-refreshes via TokenSource
  - Retry loop with exponential backoff (1s, 2s, 4s) for 429/500/503 responses
  - Verbose mode logs full request (method, URL, headers with redaction, body) and response (status, timing) to stderr
- `internal/api/errors.go` ‚Äî NEW: API error parsing and mapping
  - `ParseAPIError(*http.Response)` ‚Äî parses Google Ads REST error envelope into `APIError`
  - Maps known error types (AUTHENTICATION_ERROR, AUTHORIZATION_ERROR, QUOTA_ERROR, QUERY_ERROR, REQUEST_ERROR) to human-readable messages
  - Falls back to HTTP status mapping for unstructured errors
- `cmd/root.go` ‚Äî added `--retries` flag (default 3), bound to viper key `retries`

### 2026-03-18 ‚Äî Phase 1.5 Output Formatters
